# PrivacyGuard Cursor 迭代包

本目录包含一套基于现有 `PrivacyGuard` 项目框架整理的 Markdown 文件，目标是：

1. **在最小改动前提下补全项目缺失部分**；
2. **保持高内聚、低耦合**；
3. **适配 Cursor 多轮迭代生成代码**；
4. **优先保证项目框架先搭建完整，再逐步补实实现细节**。

## 文件列表

- `01_PrivacyGuard_项目准则与迭代总计划.md`
- `02_第1轮_项目骨架与接口契约.md`
- `03_第2轮_OCR与PII检测链路.md`
- `04_第3轮_Persona仓库与会话映射.md`
- `05_第4轮_决策引擎与约束解析.md`
- `06_第5轮_渲染引擎与还原闭环.md`
- `07_第6轮_API整合_测试与交付.md`

## 建议使用顺序

1. 先把 `01_PrivacyGuard_项目准则与迭代总计划.md` 作为项目总纲读完；
2. 按编号顺序把第 1~6 轮文件逐轮贴给 Cursor；
3. 每轮只允许 Cursor 实现当前轮涉及的文件，不要跨轮大改；
4. 每轮完成后先本地运行测试，再进入下一轮；
5. 若某轮实现与前一轮契约冲突，优先回退到“接口不变，内部补充”的方式修正。

## 默认技术栈

本迭代包默认项目使用 **Python 3.11+**，并以如下库为主：

- `pydantic`：结构化数据模型
- `Pillow`：图像重绘
- `numpy`：图像数组处理
- `pytest`：测试
- `pyyaml`：配置
- `typing_extensions`：类型增强

OCR 与 NER 只先完成**适配器层与可替换接口**，其中：

- OCR 默认对接 **PP-OCRv5**
- 轻量 NER 默认对接 **GLiNER PII Edge**
- `llm_based` 检测模式只保留插件接口，不作为首轮刚需

## 交付原则

- 先搭框架，后补算法；
- 先打通 API_1 / API_2 闭环，后优化策略质量；
- 先用可恢复、可测试、可替换的保守实现，后再增强模型能力；
- 不引入不必要的复杂基础设施（消息总线、微服务、数据库集群等）。
