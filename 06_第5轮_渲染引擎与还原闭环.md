# 第 5 轮：渲染引擎与还原闭环

## 可直接粘贴到 Cursor 的简短提示词

请只实现 **Rendering Engine** 与 **Restoration Module**，目标是把决策计划真正应用到 prompt 和 screenshot 上，并能在 API_2 中把云端返回文本恢复为真实值。  
严格不要在本轮大改前面几轮的接口，只在现有契约上补齐闭环。

---

## 本轮目标

完成以下闭环：

1. 输入 `DecisionPlan + PIICandidate + 原始 prompt + 原始 screenshot`
2. 产出脱敏后的 prompt 和 screenshot
3. 写入 `ReplacementRecord`
4. 接收云端返回文本
5. 根据 mapping 恢复为真实文本

---

## 必须遵守的约束

1. Rendering Engine 只做“应用替换”，不重新决策；
2. Restoration Module 只做“按映射恢复”，不新增替换；
3. prompt 替换与 screenshot 替换必须共享同一替换记录；
4. 图像重绘优先保证“可用、可对齐、可恢复”，不先追求美观极致；
5. 恢复规则必须明确、可测试、可解释。

---

## Prompt Renderer 要求

### 功能
- 按 `DecisionPlan.actions` 对 `prompt_text` 做替换
- 生成替换后的文本
- 生成与文本替换对应的 `ReplacementRecord`

### 规则
- 替换顺序要可控，避免短串先替换导致长串错位；
- 建议采用“按原文本长度倒序 + 明确边界”策略；
- 对完全相同但不同语义的文本要谨慎替换，必要时结合候选来源与位置。

### 输出
建议返回：
- `sanitized_prompt_text`
- `applied_records`

---

## Screenshot Renderer 要求

### 功能
- 根据 OCR bbox 定位原文区域
- 对区域进行覆盖与重绘
- 输出新的 screenshot

### 最小可行实现建议
1. 使用 Pillow
2. 先用背景色近似覆盖原区域
3. 再用替代文本重绘
4. 字号允许做近似，不追求完全一致

### 需要考虑
- bbox 为空时跳过图像替换
- replacement_text 过长时做保守策略：
  - 截断
  - 缩小字号
  - metadata 中记录 warning
- 不要在这一轮实现复杂版式引擎

---

## sanitize_pipeline 要求

请在 pipeline 中按固定顺序串联：

1. OCR
2. PII Detector
3. Session State Read
4. Decision Engine
5. Prompt Renderer
6. Screenshot Renderer
7. Mapping Store Write
8. 组装 `SanitizeResponse`

### 原则
- pipeline 只负责流程编排；
- 具体逻辑仍在各模块内部；
- 不要把所有逻辑塞进 pipeline。

---

## Restoration Module / action_restorer 要求

目标是把云端返回文本中的：

- `<NAME>`
- `<PHONE>`
- persona 假值
- 其他替代文本

恢复成真实值。

### 建议恢复策略
1. 优先按当前 turn mapping 精确恢复
2. 若当前 turn 找不到，再查当前 session 历史记录
3. 若存在歧义，优先：
   - 更近 turn
   - 完整匹配
   - attr_type 一致记录

### 要求
- 恢复过程必须是**保守的**
- 找不到映射时不要瞎恢复，保留原文本并记录 metadata

---

## restore_pipeline 要求

按以下顺序：

1. 读取 session / turn 对应 mapping
2. 调用 `action_restorer`
3. 输出 `RestoreResponse`

必要时可以支持：
- 当前 turn 优先
- session 全局回溯

---

## Roundtrip 集成测试要求

请至少写一个端到端小测试，验证：

- 输入 prompt：包含姓名、手机号
- 输入 OCR：包含地址
- 使用 `label_persona_mixed`
- 经过 sanitize 后文本被替换
- 记录写入 mapping store
- 模拟云端返回包含替代值
- restore 后能还原为真实值

---

## 禁止事项

- 不要重写 Decision 逻辑；
- 不要在 Restoration 里临时重新做 PII 检测；
- 不要把图像渲染和 prompt 替换混成一个大类；
- 不要为了通过测试把恢复逻辑写成纯硬编码字符串替换。
