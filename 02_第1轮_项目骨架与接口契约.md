# 第 1 轮：项目骨架与接口契约

## 可直接粘贴到 Cursor 的简短提示词

请基于 PrivacyGuard 现有架构，只完成**项目骨架、领域模型、抽象接口、DTO、工厂注册机制**的搭建。  
严格限制在“可导入、可实例化、可测试的框架层”，不要提前实现 OCR/PII/渲染/还原的完整业务逻辑，不要跨轮开发。

## 本轮目标

本轮只做一件事：**把项目的静态骨架与动态装配能力搭起来**，让后续所有模块都有稳定落点。

完成后应达到：

- 项目目录结构清晰；
- `domain` 中有统一数据模型与枚举；
- `api` 中有 API_1 / API_2 的 DTO；
- `domain.interfaces` 中有所有核心模块的抽象接口；
- `bootstrap` 中有工厂与注册器；
- `facade` 可以被实例化；
- 即使很多实现还是占位，也能跑最基本的导入测试。

---

## 必须遵守的约束

1. 不实现具体 OCR、PII 检测、Decision、Rendering、Restoration 逻辑；
2. 只定义接口、基础空实现、异常、DTO、领域模型；
3. 所有数据对象统一使用 `pydantic` 或 dataclass，但全项目必须统一一种主要风格；
4. 外层模块只能依赖接口，不直接依赖具体实现；
5. 不允许在一个文件里塞过多类别，按职责拆文件；
6. 不要引入数据库、消息队列、Web 框架。

## 接口设计要求

每个接口只保留最小必要方法。例如：

### OCR
```python
class OCREngine(Protocol):
    def extract(self, image: ImageLike) -> list[OCRTextBlock]: ...
```

### PII Detector
```python
class PIIDetector(Protocol):
    def detect(self, prompt_text: str, ocr_blocks: list[OCRTextBlock]) -> list[PIICandidate]: ...
```

### Persona Repository
```python
class PersonaRepository(Protocol):
    def get_persona(self, persona_id: str) -> PersonaProfile | None: ...
    def list_personas(self) -> list[PersonaProfile]: ...
    def get_slot_value(self, persona_id: str, attr_type: str) -> str | None: ...
```

### Mapping Store
```python
class MappingStore(Protocol):
    def save_replacements(self, session_id: str, turn_id: int, records: list[ReplacementRecord]) -> None: ...
    def get_replacements(self, session_id: str, turn_id: int | None = None) -> list[ReplacementRecord]: ...
    def get_session_binding(self, session_id: str) -> SessionBinding | None: ...
    def set_session_binding(self, binding: SessionBinding) -> None: ...
```

其他接口也遵循“最少方法、面向职责”的原则。

## facade 要求

`facade.py` 提供统一入口类，例如：

```python
class PrivacyGuardFacade:
    def sanitize(self, request: SanitizeRequest) -> SanitizeResponse: ...
    def restore(self, request: RestoreRequest) -> RestoreResponse: ...
```

注意：

- 当前轮允许内部抛 `NotImplementedError`；
- 但 `PrivacyGuardFacade` 必须能正确装配依赖；
- 初始化方式尽量支持配置注入。

---

## factories / registry 要求

### registry
维护类似下列映射：

- OCR provider 名称 -> OCR 实现类
- detector mode -> detector 实现类
- decision mode -> decision 实现类
- mapping store type -> mapping 实现类
- persona repository type -> repository 实现类

### factories
负责：

1. 读取配置；
2. 从 registry 选择实现；
3. 创建实例；
4. 把实例注入给 facade/pipeline。

当前轮如果具体实现尚不存在，可以先注册占位实现类。
