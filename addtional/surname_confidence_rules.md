# 中文姓氏单字置信度规则说明

## 目标

这份文档用于指导代码实现中文姓名中的“姓氏首字置信度”规则。  
它不是按人口数量排序，而是按**单字在普通文本中的歧义强弱**来分层，目标是服务于：

- OCR 后处理
- NER 后处理
- 姓名候选打分
- 假阳性抑制
- 复姓优先匹配

核心原则：

1. **复姓优先于单姓**
2. **大姓不等于强单字信号**
3. **高频实词、功能词、方向词、颜色词、常见名词不能因为属于百家姓就给高权重**
4. **有些中等姓虽然普通用法不少，但主要集中在有限固定词组里；排除这些词组后，可以拔高其单字置信度**

---

## 推荐实现顺序

1. 先扫描复姓，采用**最长优先、精确匹配**
2. 如果未命中复姓，再处理单姓
3. 单姓先看 `strong / medium / weak`
4. 对 `boostable_medium_surnames`：
   - 若命中对应黑名单固定词组，则不拔高
   - 若未命中，则从 `medium` 临时提升到 `boostable_medium_after_blacklist_pass`
5. 再结合位置、字段语义、后接名用字做最终总分

---

## 建议打分

```json
{
  "compound_exact_match": 6,
  "single_strong": 3,
  "single_medium": 1,
  "single_weak": -2,
  "boostable_medium_after_blacklist_pass": 2,
  "position_first_char_of_2_to_3_char_cjk_span": 2,
  "common_given_name_char_after_surname": 2,
  "name_field_or_contact_context": 3,
  "fixed_phrase_blacklist_hit": -4,
  "non_name_field_context": -2
}
```

> 这些分值不是唯一正确答案，但适合作为第一版落地实现的默认值。

## 复姓规则

### 匹配策略

- 只允许**整串精确匹配**
- 必须**先于单姓**
- 必须**最长优先**
- 不能只看复姓的首字

### 复姓表

```text

```

## 单姓分层

## 1. 强姓氏信号字

定义：  
这些字单独出现时，比一般字更像姓氏；普通文本中的强势实词功能相对较弱。

## 2. 中等姓氏信号字

定义：  
这些字经常作为姓氏出现，但同时也具有一定普通词汇功能。  
默认不能单字直判，必须结合上下文。

```text

```

## 3. 容易误判字

定义：  
这些字虽然在百家姓里，但它们在现代汉语里本身就是高频通用字或高频实词。  
**禁止仅凭单字就高置信判为姓氏。**

```text
李 周 王 陈 何 张 孔 严 华 金 水 章 云 苏 范 马 花 方 任 史 唐 汤 安 常 乐 于 时 傅 余 元 平 黄 和 毛 米 明 计 成 纪 舒 梁 杜 强 路 江 林 高 夏 田 胡 万 管 经 干 宗 丁 洪 石 程 陆 家 山 车 侯 宁 甘 斜 武 景 龙 叶 司 白 从 乔 能 别 鱼 容 向 古 易 终 居 步 都 满 国 文 广 东 童
```

典型例子：

- 王：王者、国王
- 周：周一、一周、周边
- 张：张开、一张、张贴
- 陈：陈列、陈述
- 方：方向、方法、方面
- 高：高低、高校、高速
- 夏：夏天、夏季
- 叶：叶子、树叶
- 童：儿童、童年、童装

## 可拔高的中等姓

定义：  
这类字默认只属于 `medium`。  
但它们在普通文本中的非姓名用法通常集中在少量固定词组里。  
**只要排除了这些固定词组，就可以把该字从 `medium` 临时提升一层。**

### 可拔高姓氏表


## 实现建议

## 1. 复姓必须先做

错误做法：

- 先把 `欧阳` 拆成 `欧` + `阳`
- 先把 `司马` 拆成 `司` + `马`

正确做法：

- 先扫复姓
- 命中后直接作为一个整体姓氏单元

---

## 2. 单姓只提供“姓氏先验”，不要直接当最终结论

建议公式：

```text
total_score =
  surname_base_score
+ span_position_score
+ following_given_name_score
+ field_context_score
- fixed_phrase_penalty
- non_name_context_penalty
```

---

## 3. 强层也不是无条件命中

即使是 `strong`，也仍然建议配合下面几类证据：

- 处于 2~3 字中文串首位
- 后接常见名用字
- 所在字段是姓名、联系人、收件人、患者、住户、客户等
- 前面出现称谓或人名提示词

---

## 4. 弱层禁止单字直判

这层字最容易误报。  
例如 `王 / 周 / 张 / 陈 / 方 / 高 / 夏 / 叶 / 童`，如果没有明显姓名上下文，默认不应高分触发。

---

## 5. 关于多音姓

下面这些姓建议在实现里单独留注释，避免后续误处理：

- 乐
- 单
- 解
- 仇

这份配置只处理“是否更像姓氏字”，不处理读音判定。

---

## 6. 适合接入代码的字段建议

建议在规则文件中保留以下结构：

- `compound_surnames`
- `single_surname_tiers.strong`
- `single_surname_tiers.medium`
- `single_surname_tiers.weak`

## 7. 推荐调用顺序伪代码

```python
def score_surname(candidate_span, field_name=None, left_context="", right_context=""):
    # 1. 复姓优先
    compound = longest_compound_match(candidate_span)
    if compound:
        score = SCORE["compound_exact_match"]
        return score, compound

    # 2. 单姓首字
    ch = candidate_span[0]
    if ch in STRONG:
        score = SCORE["single_strong"]
    elif ch in MEDIUM:
        score = SCORE["single_medium"]
        if ch in BOOSTABLE and not hit_blacklist_phrase(ch, left_context, right_context):
            score += SCORE["boostable_medium_after_blacklist_pass"]
    elif ch in WEAK:
        score = SCORE["single_weak"]
    else:
        score = 0

    # 3. 位置与上下文
    if is_first_char_of_2_to_3_char_cjk_span(candidate_span):
        score += SCORE["position_first_char_of_2_to_3_char_cjk_span"]

    if has_common_given_name_char_after_surname(candidate_span):
        score += SCORE["common_given_name_char_after_surname"]

    if is_name_field(field_name) or has_name_context(left_context, right_context):
        score += SCORE["name_field_or_contact_context"]

    if is_non_name_field(field_name):
        score += SCORE["non_name_field_context"]

    return score, ch
```

## 8. 这一版配置的定位

这是**工程优先**的第一版规则，不是语言学终局答案。  
后续建议通过你的真实 OCR / detector 输出做误报统计，再微调：

- 强层是否需要收缩
- 中等层是否需要扩展
- 黑名单词组是否要按业务场景增加
- 不同字段下是否要有不同分值
